Logger가 있는 클래스가 아닌클래스 리터럴을 사용해 초기화된 Logger 인스턴스를 보고합니다.

이는 다른 클래스에서 복사하여 붙여넣기할 때 일어날 수 있습니다. 그리고 이로 인해 예상하지 못한 카테고리 및 부정확한 필터링이 이벤트 기록 시 적용될 수 있습니다.

이 검사에서 인식되는 로거 factory 클래스 및 메서드를 지정하려면 검사 옵션을 사용하세요.

예:


class AnotherService
class MyService {
    private val logger = LoggerFactory.getLogger(AnotherService::class.java)
}

빠른 수정 적용 후:


class MyService {
    private val logger = LoggerFactory.getLogger(MyService::class.java)
}
