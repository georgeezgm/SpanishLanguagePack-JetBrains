대입 후 사용되지 않은 대입 값을 보고합니다. 대입 값이 사용되지 않을 경우 대입을 제거하여 코드 길이를 줄이고 불필요한 대입을 피하는 것이 좋습니다.

다음과 같은 경우가 보고됩니다:

  • 변수가 대입된 후 한 번도 읽히지 않은 경우
  • 변수가 읽히기 전에 항상 새 값으로 덮어쓰이는 경우
  • (위의 두 가지 이유 중 하나로) 변수 이니셜라이저가 불필요한 경우

검사 구성:

Report redundant initializers(불필요한 이니셜라이저 보고) 옵션을 사용하여 불필요한 이니셜라이저를 보고합니다. 


  int getI() {
    int i = 0; // 불필요한 이니셜라이저
    i = 2;
    return i;
  }

Report ++i when may be replaced with (i + 1)((i + 1)로 대체 가능할 시 ++i 보고) 옵션을 사용하여 ++i 표현식을 i + 1로 대체 가능한 경우를 보고합니다. 


  int preInc(int i) {
    int res = i;
    return ++res;
  }

Report i++ when changed value is not used afterwards(추후 변경된 값이 사용되지 않을 시 i++ 보고) 옵션을 사용하여 i++ 표현식의 결과가 추후 사용되지 않는 경우를 보고합니다. 


  int postInc(int i) {
    int res = i;
    return res++;
  }
