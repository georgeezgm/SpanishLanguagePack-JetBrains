대입을 통한 필드 초기화를 필드 이니셜라이저로 바꿀 것을 제안합니다.

필드 이니셜라이저가 이니셜라이저 내에 위치하는 경우에만 코드가 강조 표시되며, 이를 필드 선언과 결합하는 것은 대체로 안전합니다. 다른 경우, 검사는 코드 강조 표시 없이 빠른 수정만을 제공하며 수정 사항은 코드 의미를 변경할 수 있습니다.

예:


  class MyClass {
    static final int intConstant;
    
    static {
      intConstant = 10;
    }
  }
빠른 수정은 대입된 값을 가능한 경우 클래스 이니셜라이저를 제거할 수 있는 필드 이니셜라이저로 이동합니다. 

  class MyClass {
    static final int intConstant = 10;
  }
