super를 호출하지 않고 특정 메서드를 재정의하는 메서드를 보고합니다.

이는 상속 거부라고도 합니다. 이러한 메서드는 추상화 실패를 의미할 수 있으며, 이로 인해 추적이 어려운 버그가 발생할 수 있습니다.

이 검사는 디폴트 메서드 및 clone()을 제외한 java.lang.Object에서 재정의된 메서드를 보고하지 않습니다. clone() 메서드는 super를 호출해야 하며, 이는 올바른 유형의 객체를 자동으로 반환합니다.

예:

  • 
  class A {
    @Override
    public Object clone() { // 'super.clone()'을 호출하지 않으므로 보고됩니다
      return new A();
    }
  }
  
  • 
  interface I {
    default void foo() {}
  }

  class A implements I {
    // ''default' 상위 메서드 무시' 및 '어노테이션이 추가된 경우 무시' 옵션이 비활성화되면 메서드에 대해 경고합니다
    @Override
    public void foo(){}
  }

검사 구성:

  • 상위 메서드에 다음으로 어노테이션이 추가된 경우에만 보고 옵션을 사용하여 제공된 목록의 어노테이션이 표시된 super 메서드를 무시합니다. 목록에 어노테이션을 수동으로 추가할 수 있습니다.
  • 빈 상위 메서드 무시 옵션을 사용하여 비어 있거나 예외만을 던지는 super 메서드를 무시합니다.
  • 'default' 상위 메서드 무시 옵션을 사용하여 default 키워드가 있는 super 메서드를 무시합니다.
