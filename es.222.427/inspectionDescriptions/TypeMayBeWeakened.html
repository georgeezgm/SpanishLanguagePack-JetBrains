더 추상적인(약한) 타입으로 변경 가능한 변수 및 메서드 반환 타입을 보고합니다. 이를 통해 더 추상적이며 재사용이 한층 용이한 코드를 만들 수 있습니다.

예:


  // 매개변수 타입을 java.util.List로 약화할 수 있습니다
  void processList(ArrayList<String> list) {
    if (list.isEmpty()) return;
    System.out.println("Processing");
    for (String s : list) {
      System.out.println("String: " + s);
    }
  }

하단의 오른쪽 타입 사용 체크박스를 활성화하여 대입의 오른쪽이 형 변환이나 새 표현식이 아닐 경우 대입의 왼쪽을 약화하지 않도록 방지합니다. 메서드 호출 결과를 변수에 저장할 경우 메서드 호출 결과의 타입을 불필요하게 약화하는 대신 유지하는 것이 유용합니다.

하단의 매개변수화된 타입 사용 체크박스를 활성화하면 매개변수 타입이 java.lang.Object인 컬렉션 메서드의 인수로 평가된 객체를 사용할 경우, 매개변수화된 컬렉션 타입을 가장 약한 타입으로 사용합니다. 이 옵션을 사용하면 get(), remove(), contains(), indexOf(), lastIndexOf(), containsKey() 및 containsValue()와 같은 컬렉션 메서드로 객체를 전달할 때 Object로 약화되지 않도록 방지할 수 있습니다.

하단의 Object로 약화하지 않음 체크박스를 활성화하면 타입을 java.lang.Object로 약화할지 지정합니다. java.lang.Object로 약화가 유용한 경우는 거의 없습니다.

하단의 인터페이스로만 약화 체크박스를 활성화하면 해당 타입이 인터페이스 타입으로 약화될 수 있는 경우에만 문제를 보고합니다.

하단의 반환 타입을 약화하지 않음 체크박스를 활성화하면 반환 타입이 약화될 수 있는 경우 문제를 보고하지 않도록 방지합니다. 변수만 분석됩니다.

하단의 'var'로 선언된 변수 약화를 제안하지 않음 체크박스를 활성화하여 'var' 키워드를 사용해 선언된 지역 변수를 보고하지 않도록 방지합니다(Java 10 이상).

stop 클래스는 가능한 경우 stop 클래스보다 낮은 클래스로 약화하지 않도록 방지합니다. 일부 경우엔 이를 통해 가독성을 개선할 수 있습니다.
