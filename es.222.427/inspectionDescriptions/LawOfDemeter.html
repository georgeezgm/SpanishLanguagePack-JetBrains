데메테르의 법칙을 위반하는 경우를 보고합니다.

데메테르의 법칙은 실제로 법칙은 아니지만, 스타일 가이드라인을 지정합니다. 다른 호출로부터 받은 객체에서 메서드를 호출하지 않습니다. 이 가이드라인을 따르는 코드는 유지 관리, 조정, 리팩터링이 쉬우며 메서드 간의 결합 및 중복이 적고 정보 숨기기가 뛰어납니다. 한편, 이 가이드라인을 준수하려면 여러 래퍼 메서드를 작성해야 합니다.

예:


  boolean pay(Customer c, Invoice invoice) {
    int dollars = c.getWallet().getDollars(); // violation
    if (dollars >= invoice.getAmount()) {
      Wallet w = c.getWallet();
      w.subtract(invoice.getAmount()); // violation
      return true;
    }
    return false;
  }
위의 예는 Customer의 payInvoice(Invoice invoice) 메서드로 더 잘 구현될 수 있습니다다.
