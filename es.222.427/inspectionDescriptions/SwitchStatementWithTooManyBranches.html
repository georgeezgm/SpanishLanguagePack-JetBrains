case 라벨이 너무 많은 switch 문 또는 식을 보고합니다.

그러한 긴 switch 문은 혼란을 일으킬 수 있으므로 리팩터링이 필요할 수 있습니다. 이러한 구문이 문제가 아닌 경우도 있습니다(예: 도메인이 매우 복잡하며 상수가 많은 열거형을 포함하는 경우).

예:


    switch (x) {
        case 1 -> {}
        case 2 -> {}
        case 3 -> {}
        case 4 -> {}
        case 5 -> {}
        case 6 -> {}
        case 7 -> {}
        case 8 -> {}
        case 9 -> {}
        case 10 -> {}
        case 11,12,13 -> {}
        default -> {}
    }

최대 브랜치 수 필드를 사용하여 필요한 case 라벨의 최대 개수를 지정하세요.
