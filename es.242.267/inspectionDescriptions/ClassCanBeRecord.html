레코드 클래스로 변환될 수 있는 클래스를 보고합니다.

레코드 클래스는 확장 가능한 동작보다는 불변 데이터를 모델링하는 데 주안점을 둡니다. equals() 및 접근자와 같이 데이터 기반 메서드의 묵시적 자동 구현은 보일러플레이트 코드를 줄이는 데 유용합니다.

모든 클래스가 레코드 클래스가 될 수 있는 것은 아닙니다. 다음 몇 가지 제한 사항입니다:

  • The class must have no subclasses.
  • 클래스 내 모든 비 정적 필드는 final이여야합니다.
  • 이니셜라이저, 제네릭 생성자 및 네이티브 메서드가 존재해서는 안됩니다.

레코드 클래스의 자세한 설명은 Java 언어 사양을 참조하세요.

예:


  class Point {
    private final double x;
    private final double y;

    Point(double x, double y) {
      this.x = x;
      this.y = y;
    }

    double getX() {
      return x;
    }

    double getY() {
      return y;
    }
  }

빠른 수정 적용 후:


  record Point(int x, int y) {
  }

getX()/isX() 접근자 이름을 x()로 자동 변경하게 하려면 Suggest renaming accessors(접근자 이름 변경 제안) 옵션을 활성화하세요.

If members become more accessible(멤버의 접근성이 개선되는 경우) 옵션을 사용하여 변환 후 멤버의 접근성이 높아지는 경우의 동작을 지정할 수 있습니다.

  • 접근성이 더 높아지는 경우 멤버를 변환하지 않도록 하려는 경우 Do not suggest conversion(변환 제안하지 않음)을 선택하세요.
  • 영향을 받는 멤버를 표시하고 계속할지 물어보도록 하려는 경우 Show conflicts view(충돌 뷰 표시) 옵션을 선택하세요. 배치 모드에서는 변환이 제안되지 않습니다.
  • 필요 시 접근성을 조용히 높이려는 경우 Convert silently(조용히 변환) 옵션을 선택하세요.

지정된 패턴과 일치하는 어노테이션이 추가된 경우, 변환에서 클래스를 제외하려면 Suppress conversion if class is annotated by(클래스에 다음 어노테이션이 추가된 경우 변환 억제) 목록을 사용하세요.
