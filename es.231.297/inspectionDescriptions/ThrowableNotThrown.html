생성된 Throwable이 실제로 한 번도 던져지지 않는 경우, Throwable 또는 그 하위 클래스의 인스턴스화를 보고합니다. 또한 이 검사는 메서드 호출의 결과가 던져지지 않는 경우 Throwable 또는 그 하위 클래스의 인스턴스를 반환하는 메서드 호출을 보고합니다.

Error Prone 또는 AssertJ의 @CanIgnoreReturnValue 어노테이션이 추가된 메서드 호출은 보고되지 않습니다.

예:


    void check(String s) {
        if (s == null) {
            new NullPointerException("s");
        }
        // ...
    }
