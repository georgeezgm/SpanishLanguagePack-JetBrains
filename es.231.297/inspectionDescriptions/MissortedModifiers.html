제어자가 표준 선호 순서(Java 언어 사양에 명시된 대로)로 되어 있지 않은 선언을 보고합니다.

예:


  class Foo {
    native public final void foo();
  }

빠른 수정 적용 후:


  class Foo {
    public final native void foo();
  }

검사 설정을 사용하여 다음을 수행합니다.

  • 잘못 배치된 어노테이션(타입 바로 앞에 그리고 제어자 키워드 뒤에 ElementType.TYPE_USE가 없는 어노테이션 또는 제어자 키워드 앞에 없는 기타 어노테이션)에 대한 보고를 전환합니다. 이 옵션이 비활성화된 경우, 어노테이션을 제어자 키워드 앞 또는 뒤에 배치할 수 있습니다. 제어자 키워드 사이에 어노테이션이 있는 제어자 목록은 항상 보고됩니다.

  • ElementType.TYPE_USE 어노테이션에 다른 대상이 지정되어 있는 경우에도 이 어노테이션을 타입 바로 앞에 배치할지 여부를 지정합니다.
