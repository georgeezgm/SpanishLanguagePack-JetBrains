..< 연산자로 바꿀 수 있는 until 사용 위치를 보고합니다.

until을 ..< 바꾸는 어떠한 경우에도 의미가 변하지 않습니다.

UX 조사 결과 개발자들이 until보다 ..<을 포함한 코드를 읽을 때 오류를 20-30% 더 적게 만드는 것으로 밝혀졌습니다.

예:


  fun main(args: Array<String>) {
      for (index in 0 until args.size) {
          println(index)
      }
  }

빠른 수정 적용 후:


  fun main(args: Array<String>) {
      for (index in 0..<args.size) {
          println(index)
      }
  }

이 검사는 프로젝트나 모듈의 코틀린 언어 수준이 1.7 이상일 때만 보고하며, -opt-in=kotlin.ExperimentalStdlibApi 컴파일러 플래그가 제공됩니다.
