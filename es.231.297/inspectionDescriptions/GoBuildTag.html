형식이 잘못된 빌드 태그 및 잘못된 위치의 빌드 태그를 보고합니다. go 도구는 빌드 태그가 특정 위치에 있어야 하며 특정한 구문을 따라야 합니다. 이러한 필수 사항을 따르지 않는 경우 빌드 태그가 무시되거나 빌드에서 파일이 잘못 제외될 수 있습니다.

go.dev의 Build Constraints(을)를 참고하세요.

예:

package main

// +build ignore

func main() {}

// +build ignore 부분이 패키지 선언 앞에 와야합니다. 이를 수정하려면 빌드 태그를 패키지 앞에 위치 빠른 수정을 적용하세요. 빠른 수정 적용 후:

// +build ignore

package main

import "fmt"
