권장되는 이름 생성 규칙을 따르지 않는 private 프로퍼티 이름을 보고합니다.

이름을 생성할 때 일관성을 유지하면 코드를 읽고 이해하기 쉽습니다. Kotlin 공식 스타일 가이드에 따르면 private 프로퍼티 이름은 소문자로 시작해야 하고 낙타 표기법을 사용해야 합니다. 밑줄 접두사를 사용하는 옵션도 있지만 private 프로퍼티에만 가능합니다.

'Pattern' 정규 표현식을 변경하면 다른 이름 생성 규칙도 사용할 수 있습니다.

예:


  val _My_Cool_Property = ""

빠른 수정에서는 Kotlin 이름 생성 규칙에 따라 이름을 변경합니다.


  val _myCoolProperty = ""
