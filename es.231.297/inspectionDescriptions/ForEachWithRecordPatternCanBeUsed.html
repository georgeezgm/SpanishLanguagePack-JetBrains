Reports local variable declarations and accessors to record components that can be replaced with pattern variables in enhanced `for` statements, which are usually more compact.

예:


record Record(Integer x, String y) {
}

public static void test(List<Record> records) {
    for (Record record : records) {
        System.out.println(record.y());
        Integer x = record.x;
        System.out.println(x);
    }
}

다음으로 대체 가능:


record Record(Integer x, String y) {
}

public static void test(List<Record> records) {
    for (Record(Integer x, String y) : records) {
        System.out.println(y);
        System.out.println(x);
    }
}

이 검사는 프로젝트 또는 모듈의 언어 버전이 20 이상인 경우에만 보고합니다

  • Enable the Use 'var' instead of explicit types option to use var in deconstruction patterns
  • Use the Nesting depth limit option to specify the allowed maximum number of nested deconstruction patterns to report
  • Use the Maximum number of record components to deconstruct option to specify the maximum allowed number of components, which a record can contain to be used in deconstruction patterns
  • Use the Maximum number of not-used record components option to specify the allowed maximum number of components, which are not used in for statement
