HCL 블록 내에서 충돌하는 프로퍼티를 보고합니다.

문제 예시:


provider "kubernetes" {
  version = "~> 1.25"
  config_path = "~/.kube/config"
  version = "~> 1.24"
}

이 예시에서 버전 프로퍼티는 Kubernetes 제공자 블록에서 두 번 선언되며 이는 오류입니다.

수정 방법 예시:


provider "kubernetes" {
  version = "~> 1.25"
  config_path = "~/.kube/config"
}
