유사한 로그 메시지를 가지며 logger.info("message: {}", key)와 같은 SLF4J, Log4j2 로그 호출이 한 클래스에 있는 경우 보고합니다. 이러한 호출은 서로 분간하기가 불가능할 수 있으며 특정 로그 메시지의 출처가 어디인지 파악하기 어려울 수 있습니다.

예(Java):


  private static void request1(String text) {
      log.info("Message: {}", text); //유사한 호출
      doSomething1();
  }

  private static void request2(int i) {
      log.info("Message: {}", i); //유사한 호출
      doSomething2();
  }
  • Minimum length of a similar sequence(유사한 시퀀스의 최소 길이) 옵션을 사용하여 호출이 보고되는 유사 시퀀스의 최소 길이를 지정하세요
  • Do not report calls with the 'error' log level('error' 로그 레벨이 있는 호출은 보고하지 않음) 옵션을 사용하면 메시지에 `error` 로그 레벨이 있거나 예외가 있는 경우 무시합니다. 호출 위치는 여전히 스택 추적으로 찾을 수 있기 때문에 경고를 숨기는 데 유용합니다.
