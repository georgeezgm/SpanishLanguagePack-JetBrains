예를 들어 @Suppress("HEADER_WITHOUT_IMPLEMENTATION") 같은 오래된 진단 이름을 가진 억제를 보고합니다.

Kotlin 1.2 또는 이전 버전의 일부 진단은 이제 곧 사용할 수 없게 되므로 그러한 억제는 불필요해집니다.

예:


@Suppress("HEADER_DECLARATION_WITH_BODY")
expect fun connection() {
  // ...
}

빠른 수정 적용 후:


@Suppress("EXPECTED_DECLARATION_WITH_BODY")
expect fun connection() {
  // ...
}
