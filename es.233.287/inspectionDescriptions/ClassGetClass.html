java.lang.Class 인스턴스에서 호출되는 getClass() 메서드를 보고합니다.

결과는 항상 Class.class와 동일하므로, 이런 경우는 대개 실수입니다. 실수인 경우 getClass() 호출을 제거하고 한정자를 직접 사용하는 것이 좋습니다. 의도된 동작인 경우 혼동을 피하기 위해 Class.class를 명시적으로 작성하는 것이 좋습니다.

예:


  void test(Class<?> clazz) {
    String name = clazz.getClass().getName();
  }

가능한 빠른 수정 중 하나가 적용된 후:


  void test(Class<?> clazz) {
    String name = clazz.getName();
  }
