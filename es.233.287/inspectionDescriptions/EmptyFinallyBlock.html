빈 finally 블록을 보고합니다.

일반적으로 빈 finally 블록은 코딩 오류를 의미합니다. 이러한 블록은 코드 리팩터링 후에도 남을 수 있으며 안전하게 제거할 수 있습니다.

이 검사는 JSP 파일의 빈 finally 블록을 보고하지 않습니다.

예:


  try {
    Files.readString(Paths.get("in.txt"));
  } catch (IOException e) {
    throw new RuntimeException(e);
  } finally {

  }

빠른 수정 적용 후: 


  try {
    Files.readString(Paths.get("in.txt"));
  } catch (IOException e) {
    throw new RuntimeException(e);
  }
