XML 애플리케이션 컨텍스트의 삽입점에서 일관성 없는 태그 및 속성을 보고합니다.

  • <property> 및 <constructor-arg>는 ref, value 또는 하위 요소를 지정해야 합니다
  • <property> 및 <constructor-arg>에 포함된 하위 요소가 1개를 초과할 수 없습니다
  • <ref>에는 bean 속성, local 속성 또는 parent 속성이 포함될 수 있습니다
  • <ref>는 bean을 지정해야 합니다
  • <idref>에는 bean 속성 또는 local 속성이 포함될 수 있습니다
  • <idref>는 bean을 지정해야 합니다
  • <map>는 key를 지정해야 합니다
  • <idref>에는 key 속성, key-ref 속성 또는 <key> 하위 속성이 포함될 수 있습니다

예:


  <beans>

  <!-- 1.  프로퍼티는 'ref' 속성 또는 'value' 속성 또는 하위 요소만 포함하도록 허용됩니다 -->
  <bean class="MyClass">
    <property name="bean" ref="myBean" value="foo"/>
    <!-- 프로퍼티는 'ref' 속성 또는 'value' 속성 또는 하위 요소만 포함하도록 허용됩니다 -->
  </bean>

  <!--  2.  프로퍼티가 2개 이상의 하위 요소를 포함해서는 안 됩니다 -->
  <bean class="MyFactory">
    <property name="bean">
    <ref>bean</ref> <!-- 프로퍼티가 2개 이상의 하위 요소를 포함해서는 안 됩니다 -->
     <value>foo</value> <!-- 프로퍼티가 2개 이상의 하위 요소를 포함해서는 안 됩니다 -->
     </property>
  </bean>

  <!-- 3. Map 항목은 키를 지정해야 합니다  -->
  <bean class="MyBean">
     <property name="map">
        <map>
           <entry value="foo"/>  <!-- Map 항목은 키를 지정해야 합니다 -->
        </map>
     </property>
  </bean>
  </beans>
