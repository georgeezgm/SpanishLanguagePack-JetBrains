static final로 선언되지 않은 java.lang.ThreadLocal 타입의 필드를 보고합니다.

대부분의 경우 java.lang.ThreadLocal 인스턴스는 상태를 스레드에 연결합니다. static final이 아닌 java.lang.ThreadLocal 필드는 상태를 인스턴스-스레드 결합에 연결합니다. 일반적으로 이는 불필요하며 메모리 누수와 잘못된 동작을 초래하는 버그인 경우가 많습니다.

빠른 수정은 필드를 static final로 만들도록 제안합니다.

예:


  private ThreadLocal tl = ThreadLocal.withInitial(() -> Boolean.TRUE);
