항성 null이거나 항상 null이 아닌 인수가 전달되는 Optional.ofNullable()의 사용 위치를 보고합니다. 이러한 경우에는 Optional.ofNullable()을 사용할 이유가 없습니다. Optional.empty() 혹은 Optional.of()을 사용하여 항상 비어 있거나 항싱 비어 있지는 않은 옵셔널을 생성하는 의도를 명시적으로 표시해야 합니다. 또한, Optional.ofNullable() 인수 내에 실수가 있을 수 있으므로 검사할 필요가 있습니다.

예: 


Optional<String> empty = Optional.ofNullable(null); // Optional.empty()이어야 함;
Optional<String> present = Optional.ofNullable("value"); // Optional.of("value")이어야 함;
