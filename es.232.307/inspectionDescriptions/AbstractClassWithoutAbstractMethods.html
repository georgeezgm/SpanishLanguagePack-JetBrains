abstract 메서드가 없는 abstract 클래스를 보고합니다. 대부분의 경우 abstract 메서드가 없는 abstract 클래스에는 의미가 없고, abstract 제어자를 클래스에서 제거할 수 있습니다. 인스턴스화를 방지하기 위해 클래스가 abstract로 선언된 경우, private 생성자를 사용하는 것이 더 나은 선택지인 경우가 많습니다.

예:


  abstract class Example {
    public String getName() {
      return "IntelliJ IDEA";
    }
  }

유틸리티 클래스를 무시하려면 이 옵션을 사용합니다.
