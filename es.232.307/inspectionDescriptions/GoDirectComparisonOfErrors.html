==로 오류가 직접 비교된 경우를 보고하며 대신 errors.Is를 제안합니다.

Go 1.13부터 에러는 %w 동사로 fmt.Errorf 함수를 사용하여 래핑할 수 있습니다. 따라서 상등 검사가 사용된 직접 오류 비교는 래핑된 오류로 인해 실패할 수 있습니다. Go 1.13 이후 표준 라이브러리의 errors.Is 함수를 사용하여 특정한 오류를 검사하는 것이 선호됩니다.

빠른 수정이 오류 간의 직접 비교를 errors.Is로 대체할 것을 제안합니다.

예:


if err == MyError {
  fmt.Println("MyErr")
}

빠른 수정 적용 후:


if errors.Is(err, MyError) {
  fmt.Println("MyErr")
}
