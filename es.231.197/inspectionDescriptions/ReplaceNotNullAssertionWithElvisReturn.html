elvis 연산자 및 반환(?: return)으로 바꿀 수 있는 비 null 어설션(!!) 호출을 보고합니다.

비 null 어설션은 예상하지 못한 NPE(NullPointerException)로 이어질 수 있습니다. 관례적으로 !!를 사용하지 않는 것이 좋습니다.

빠른 수정에서는 비 null 어설션을 return 또는 return null로 바꿉니다.

예:


  fun test(number: Int?) {
      val x = number!!
  }

빠른 수정 적용 후:


  fun test(number: Int?) {
      val x = number ?: return
  }
