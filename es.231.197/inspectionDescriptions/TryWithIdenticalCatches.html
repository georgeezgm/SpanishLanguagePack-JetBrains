단일 try 문 내 동일한 catch 섹션을 보고합니다.

해당 섹션을 하나의 multi-catch 블록으로 접으면, 코드 중복을 줄일 수 있을 뿐만 아니라 catch 섹션 하나는 업데이트되고 다른 하나는 업데이트되지 않는 상황을 방지할 수 있습니다.

예:


    try {
        doSmth();
    }
    catch (IOException e) {
        LOG.error(e);
    }
    catch (URISyntaxException e) {
        LOG.error(e);
    }

코드를 더 간결하게 바꾸는 빠른 수정을 이용할 수 있습니다.


    try {
        doSmth();
    }
    catch (IOException | URISyntaxException e) {
        LOG.error(e);
    }

이 검사는 프로젝트 또는 모듈의 언어 수준이 7 이상인 경우에만 보고합니다.
