동적 타입 리시버에서 asDynamic 함수의 사용을 보고합니다.

asDynamic 함수는 동적 타입의 표현식에 영향을 주지 않습니다.

동적 타입의 리시버에서 asDynamic 함수를 사용하면 asDynamic이 JavaScript 환경에서 실행되고 그러한 함수는 런타임에 없을 수 있기 때문에 런타임 문제로 이어질 수 있습니다. 이 함수는 일반적인 Kotlin 타입에서 사용하도록 의도되었습니다.

'asDynamic' 호출 제거 빠른 수정을 사용하면 코드를 자동으로 수정할 수 있습니다.

예:


  fun wrongUsage(d: Dynamic) {
     d.asDynamic().foo() // <== 불필요. 빠른 수정에서는 이 호출 표현식을 'd.foo()'로 단순화합니다
  }
