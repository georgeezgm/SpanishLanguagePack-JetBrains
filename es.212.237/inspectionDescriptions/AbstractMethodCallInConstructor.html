객체 생성 중에 현재 클래스의 abstract 메서드 호출을 보고합니다.

메서드가 다음 내부에 있는 경우 객체 생성 중에 이 메서드가 호출됩니다.

  • 생성자
  • 비 static 인스턴스 이니셜라이저
  • 비 static 필드 이니셜라이저
  • clone() 메서드
  • readObject() 메서드
  • readObjectNoData() 메서드

이러한 호출로 인해 미묘한 버그가 발생할 수 있습니다. 객체 초기화가 메서드 호출 전에 발생할 수 있기 때문입니다.

예:


  abstract class Parent {
    abstract void abstractMethod();
  }

  class Child extends Parent {
    Child() {
      abstractMethod();
    }
  }

이 검사는 다음 검사와 기능을 공유합니다.

  • 객체 생성 중에 재정의 가능한 메서드 호출
  • 객체 생성 중에 재정의된 메서드 호출

경고 중복을 피하기 위해 한 번에 하나의 검사만 활성화해야 합니다.
