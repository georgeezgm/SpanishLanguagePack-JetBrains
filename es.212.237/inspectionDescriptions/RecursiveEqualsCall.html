재귀 equals(==) 호출을 보고합니다.

== (Kotlin의 구조 비교) Java 개발자는 참조 비교로 잘못 해석할 수 있습니다. 따라서 검사에서는 이러한 사례를 보고하고 ==를 ===(Kotlin의 참조 비교)로 교체하도록 제안합니다.

예:


  class X {
      override fun equals(other: Any?): Boolean {
          if (this == other) return true
          return false
      }
  }

빠른 수정 적용 후:


  class X {
      override fun equals(other: Any?): Boolean {
          if (this === other) return true
          return false
      }
  }
