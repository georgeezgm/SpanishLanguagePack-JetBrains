동적 유형 리시버의 asDynamic 함수 사용 위치를 보고합니다.

asDynamic 함수는 동적 유형의 표현식에 영향을 주지 않습니다.

동적 유형의 리시버에서 asDynamic 함수를 사용하면 런타임 문제로 이어질 수 있는데 이는 asDynamic이 JavaScript 환경에서 실행되며 그러한 함수는 런타임에 존재하지 않을 수 있기 때문입니다. 의도한 방식은 이 함수를 일반적인 Kotlin 유형에서 사용하는 것입니다.

"asDynamic" 호출 제거 빠른 수정을 사용해 코드를 자동으로 수정할 수 있습니다.

예:


  fun wrongUsage(d: Dynamic) {
     d.asDynamic().foo() // <== 불필요, 빠른 수정이 호출 표현식을 "d.foo()"로 단순화
  }
