유연하게 null 가능한(알 수 없는 null 값 가능성이 있는 플랫폼 유형) 확장 리시버가 있는 인라인 함수의 안전하지 않을 수 있는 호출을 보고합니다.

Kotlin 1.2 미만 버전에서는 유연한 null 가능 확장 리시버(알 수 없는 null 가능성이 있는 플랫폼 유형)를 포함하는 inline fun 호출이 바이트코드의 null 값 가능성 검사에 포함되지 않았습니다. Kotlin 1.2 버전부터 null 값 가능성 검사가 바이트코드에 포함되었습니다 (KT-12899 참조).

예외가 발생하기를 원하는 명시적 !!을 추가하거나 예외가 없어도 작동하는 경우 함수의 리시버 유형을 null 가능으로 변경하는 것을 권장합니다.

빠른 수정이 적용된 후: 명시적 !!가 리시버에 추가됩니다.

이 검사는 프로젝트 또는 모듈의 Kotlin 언어 수준이 1.2보다 낮을 경우에만 보고합니다.
