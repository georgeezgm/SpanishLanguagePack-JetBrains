필드 시그니처 또는 메서드의 시그니처에 클래스가 사용되었으며 시그니처 내 클래스가 멤버 자신보다 가시성이 낮은 경우를 보고합니다. 정당한 Java이지만 그러한 멤버는 가시성 범위 밖에서는 쓸모가 없습니다.

예:

  • private 내부 class를 반환하는 public 메서드를 보고합니다.
  • 타입이 package-local 클래스인 protected 필드를 보고합니다.

Java 9에서 모듈은 패키지를 내보내기에서 제외하여 클래스의 일부를 숨길 수 있습니다. 그러므로 내보내기된 API 시그니처는 내보내기되지 않은 클래스를 포함합니다. 그러한 API는 모듈 밖에서 쓸모가 없습니다.

검사 구성:

  • 예기치 않은 클래스를 노출하는 모듈 API를 보고하려면 모듈의 API가 내보내지 않은 클래스를 노출(Java 9+) 옵션을 사용하세요. 이 옵션은 프로젝트 또는 모듈의 언어 수준이 9보다 높은 경우에 작동합니다.
  • 접근할 수 없는 클래스를 노출하는 public API를 보고하려면 공용 API가 액세스할 수 없는 클래스를 노출 옵션을 사용하세요.
  • private 클래스를 노출하는 package-local API를 보고하려면 패키지 로컬 API가 private 클래스를 노출 옵션을 사용하세요.
