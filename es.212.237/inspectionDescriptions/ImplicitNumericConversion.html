숫자 타입 간의 묵시적 변환을 보고합니다.

묵시적 숫자 변환 자체는 문제가 아니지만, 예기치 않은 경우, 버그 추적에 어려움이 발생할 수 있습니다.

예:


  double m(int i) {
    return i * 10;
  }

빠른 수정 적용 후:


  double m(int i) {
    return (double) (i * 10);
  }

검사 구성:

  • 변환 확장 무시 옵션을 사용하여 데이터 손실을 일으킬 수 없는 묵시적 변환을 무시합니다(예: int->long).
  • 'char'에서 또는 'char'로의 변환 무시 옵션을 사용하여 char 간의 변환을 무시합니다. 이 검사는 부동 소수점 수와 다른 타입 간의 변환을 계속 보고합니다.
  • 상수 및 리터럴에서의 변환 무시를 사용하여 검사에서 리터럴 및 컴파일 타임 상수의 변환을 무시하도록 만듭니다.
