단일 문자열 리터럴이 인자로 전달되는 String.startsWith() 및 String.endsWith() 호출을 보고합니다.

빠른 수정은 호출을 더 효율적으로 구현한 String.charAt() 로 치환하는 것을 권장합니다.

하지만 변경에 따른 성능 향상은 최소이며, 코드는 비 제로 길이 확인이 추가되어 가독성이 낮아집니다. 따라서 타이트한 루프 내에서만 빠른 수정을 적용하는 것을 권장합니다.

이 검사는 Java ME 및 리소스가 많이 제한된 기타 환경을 위한 것입니다. 이 검사 결과를 고려하지 않고 적용하면 코드 명확도와 디자인에 부정적인 영향을 미칠 수 있습니다.

예:


  boolean startsWithX(String s) {
    return s.startsWith("x");
  }

빠른 수정 적용 후:


  boolean startsWithX(String s) {
    return !s.isEmpty() && s.charAt(0) == 'x';
  }
