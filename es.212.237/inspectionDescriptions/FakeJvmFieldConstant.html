const가 아니고 Java 어노테이션 인수로 사용하는 Kotlin 프로퍼티를 보고합니다. 예를 들어, @JvmField로 어노테이션을 추가한 프로퍼티는 컴파일 시 평가할 수 있는 이니셜라이저가 있고 기본 또는 String 유형을 가지고 있습니다.

그러한 프로퍼티는 Kotlin 1.1~1.2의 bytecode에서 ConstantValue 속성을 가지고 있습니다. 이 속성은 javac이 대응하는 필드의 사용 위치를 접고 어노테이션에서 그 필드를 사용할 수 있도록 허용합니다. 이렇게 하면 Java/Kotlin 혼합 코드 내 개별 또는 증분 컴파일에서 부정확한 동작으로 이어질 수 있습니다. 이 동작은 Kotlin 1.3(ConstantValue 속성을 더 이상 사용하지 않음)에서 변경되기 쉽습니다. 문제를 해결하려면 관련된 Kotlin 프로퍼티에서 const 제어자가 포함된 @JvmField 어노테이션을 대체하거나 인라인화합니다.
