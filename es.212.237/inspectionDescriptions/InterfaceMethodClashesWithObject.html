java.lang.Object에서 protected 접근 제어자를 사용하는 clone() 및 finalize() 메서드와 충돌되는 인터페이스 메서드를 보고합니다.

인터페이스 내에 java.lang.Object 메서드와 호환되지 않는 반환 유형을 갖는 이러한 메서드를 선언하는 것이 가능합니다. 이는 이러한 인터페이스를 구현하는 클래스를 생성하도록 허락합니다. 인터페이스가 함수형인 경우 인터페이스에서 람다를 생성하는 것은 가능하지만 코드가 여전히 혼란을 야기할 수 있습니다.

예:


  // 경고: 이 인터페이스는 어떤 클래스에서도 구현될 수 없으며
  // 오로지 람다 또는 메서드 참조에 의해서만 구현될 수 있습니다
  interface MyInterface {
    double clone();
  }
