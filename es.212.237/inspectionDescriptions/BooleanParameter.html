boolean 매개변수를 허용하는 public 메서드를 보고합니다.

setter가 아닌 public 메서드(API의 일부)에 boolean 매개변수를 추가하는 것은 대부분의 경우 나쁜 사례입니다. 그러한 메서드를 사용하는 코드를 읽을 때 소스 또는 문서를 보지 않고는 boolean이 무엇을 의미하는지 해석하기 어렵습니다.

이 문제는 bool 함정이라고도 불립니다. boolean 매개변수는 보통 enum으로 바꿀 수 있습니다.

예:


  // 경고: 이 메서드 호출을 보고
// bool 매개변수의 의미를 이해하기 어려움
  public boolean setPermission(File f,
                               int access,
                               boolean enable,
                               boolean ownerOnly) {
    //...
  }

메서드가 하나 이상의 bool 매개변수를 사용하는 경우에만 경고하려면 bool 매개변수가 여러 개인 메서드만 보고 옵션을 사용합니다.
