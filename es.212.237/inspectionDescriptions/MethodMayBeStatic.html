안전하게 static으로 만들 수 있는 메서드를 보고합니다. 가능한 경우 메서드를 static으로 설정하면 메모리 사용량을 줄이고 코드 품질을 향상시킬 수 있습니다.

다음과 같은 경우 메서드가 static일 수 있습니다:

  • synchronized, native 또는 abstract가 아닙니다.
  • 포함 클래스의 비 static 메서드 및 비 static 필드를 참조하지 않습니다.
  • 재정의가 아니며 하위 클래스에서 재정의되지 않습니다.

다음 옵션을 사용해 검사를 구성하세요:

  • 이 검사의 성능을 향상시키기 위해, private 및 final 메서드만 보고할지 여부.
  • 빈 메서드를 무시할지 여부.
  • Java 8 이상을 사용할 때 인터페이스의 기본 메서드를 무시할지 여부.
  • static으로 설정된 메서드 호출에서 인스턴스 한정자를 클래스 참조로 빠른 수정이 바꾸도록 허용할지 여부. 즉, myClass.m() 호출이 MyClass.m()로 대체됩니다.
