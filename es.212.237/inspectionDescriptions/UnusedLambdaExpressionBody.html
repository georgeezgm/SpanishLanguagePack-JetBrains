호출한 함수가 표현식 본문에서 람다를 반환할 경우 사용하지 않은 반환 값을 가지는 호출을 보고합니다.

함수 헤더와 본문 블록 사이에 =가 있다면 함수의 코드가 평가되지 않으며 이는 부정확한 동작으로 이어질 수 있습니다.

함수 선언에서 = 토큰 제거를 사용해 코드를 자동으로 수정할 수 있습니다.

예:


  fun printHello() = { println("Hello") }

  fun main() {
      printHello() // 이 함수는 아무것도 인쇄하지 않음
  }

빠른 수정 적용 후:


  fun printHello() { println("Hello") }

  fun main() {
      printHello()
  }
