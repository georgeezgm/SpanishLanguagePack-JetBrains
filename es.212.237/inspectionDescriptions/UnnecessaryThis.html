불필요한 this 한정자를 보고합니다.

코드 참조를 명확하기 하기 위해 this 를 사용하는 것은 다양한 코드 스타일로 추천하지 않으며 자동 리팩터링을 통해 간단하게 불필요하게 될 수 있습니다.

예:

  class Foo {
    int x;
    void foo() {
      this.x = 2;
    }
  }

빠른 수정 적용 후:

  class Foo {
    int x;
    void foo() {
      x = 2;
    }
  }

검사 설정을 사용하여 필드 대입을 무시합니다. 예를 들면 this.x = 2; 는 보고되지 않지만 int y = this.x; 는 보고됩니다.
