Bean Validation <constraint-mappings> 파일 내 잘못된 요소를 보고합니다:

  • 인스턴스화할 수 없는 클래스 참조
  • 필수 클래스를 확장하지 않는 클래스 참조
  • 부적절한 스코프를 가진 클래스 참조
  • 빈 태그 및 속성 값
  • 요구되는 패턴(예: Java 식별자)과 일치하지 않는 태그 및 속성 값
  • 필요한 하위 태그 또는 속성을 포함하지 않는 태그
  • 이름이 중복된 객체를 정의하는 태그

예:


<constraint-mappings version="1.1">
    <bean class="my.UserClass" ignore-annotations="false">
        <field name="name">
            <constraint annotation="java.lang.String"/>  <!--'java.lang.String'은 Java 어노테이션이 아님 -->
        </field>

        <field name="name">
            <constraint annotation="javax.validation.constraints.Size">
                <element name="min">non-number</element>    <!--'-2,147,483,648'에서 '2,147,483,647'까지의 숫자여야 함 -->
            </constraint>
        </field>

        <getter name="zzz"> <!--심볼 'zzz'를 해결할 수 없음 -->
            <valid/>
        </getter>

    </bean>
</constraint-mappings>
