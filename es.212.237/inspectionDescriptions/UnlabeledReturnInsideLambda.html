인라인 람다에서 라벨이 없는 return 표현식을 보고합니다.

그러한 표현식은 범위가 return에 속하므로 불명확할 수 있습니다.

return@…으로 변경 빠른 수정을 사용해 코드를 자동으로 수정할 수 있습니다.

예:


  fun test(list: List<Int>) {
      list.forEach {
          // 이 반환 표현식은 함수 테스트에서 반환
          // 범위를 바꾸기 위해 이를 return@forEach로 변경 가능
          if (it == 10) return
      }
  }

빠른 수정 적용 후:


  fun test(list: List<Int>) {
      list.forEach {
          if (it == 10) return@test
      }
  }
