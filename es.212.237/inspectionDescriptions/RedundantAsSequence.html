성능에 긍정적인 영향을 주지 않는 불필요한 asSequence() 호출을 보고합니다.

일반적으로 asSequence는 컬렉션에 포함된 여러 연산을 느리게 수행하여 컬렉션의 처리 속도를 높입니다. 하지만 최종 연산(예: toList())이 asSequence() 직후에 사용될 경우 성능에 긍정적 영향을 미치지 않습니다.

예:


  fun test(list: List<String>) {
      list.asSequence().last()
  }

빠른 수정 적용 후:


  fun test(list: List<String>) {
      list.last()
  }
