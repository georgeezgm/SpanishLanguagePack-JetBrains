elvis 연산자로 바꿀 수 있고 (?: return)을 반환하는 비 null 어설션(!!) 호출을 보고합니다.

비 null 어설션은 예상하지 못한 NPE(NullPointerException)로 이어질 수 있습니다. !!을 사용하지 않는 것은 좋은 관행입니다.

빠른 수정이 비 null 어설션을 return 또는 return null으로 바꿉니다.

예:


  fun test(number: Int?) {
      val x = number!!
  }

빠른 수정 적용 후:


  fun test(number: Int?) {
      val x = number ?: return
  }
