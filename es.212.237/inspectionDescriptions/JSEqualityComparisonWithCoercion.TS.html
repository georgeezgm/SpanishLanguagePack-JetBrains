예기치 않은 타입 강제 변환을 일으킬 수 있는 상등 연산자의 사용 위치를 보고합니다. == 또는 != 상등 연산자를 타입 세이프 === 또는 !== 연산자로 바꿀 것을 제안합니다.

선택한 옵션에 따라 다음 경우 중 하나가 보고됩니다:

  • == 및 != 연산자의 모든 사용 위치.
  • null과의 비교를 제외한 모든 사용 위치. 일부 코드 스타일은 x == null을 x === null || x === undefined의 대체 코드로 사용하는 것을 허용합니다.
  • 의심스러운 표현식만(예: 0, '', null, true, false, 또는 undefined와의 == 또는 != 비교).
