null 가능 리시버에서만 의미가 있는 null이 아닌 리시버 호출을 보고합니다.

orEmpty() 또는 isNullOrEmpty 같은 표준 라이브러리의 여러 함수는 null 가능 유형의 리비서에서 호출되었을 때만 의미가 있습니다. 그렇지 않은 경우 결과가 같으므로 생략하거나 단순화할 수 있습니다.

불필요한 호출 제거 및 호출을 …로 변경 빠른 수정을 사용해 코드를 자동으로 수정할 수 있습니다.

예:


  fun test(s: String) {
      val x = s.orEmpty() // 빠른 수정이 's'로 단순화
      val y = s.isNullOrEmpty() // 빠른 수정이  's.isEmpty()'로 단순화
  }
