메서드의 시그니처에 선언되었지만 메서드 자체 또는 메서드의 구현 및 파생에서 던지지 않은 예외를 보고합니다.

이 검사는 직렬화와 관련된 메서드를 무시합니다(예: readObject() 또는 writeObject() 메서드).

예:


  void method() throws InterruptedException {
    System.out.println();
  }

빠른 수정은 선언에서 불필요한 예외를 제거하며 불필요한 try-catch 문을 정규화합니다.

  void method() {
    System.out.println();
  }

참고: 일부 예외는 에디터 내 강조 표시로 보고되지 않을 수 있습니다. 성능상의 이유로 private이 아닌 메서드는 프로젝트에서 이름이 거의 나오지 않는 경우에만 검사됩니다. 모든 결과를 확인하려면 메인 메뉴의 분석 | 코드 검사 또는 분석 | 이름별로 검사 실행을 선택하여 검사를 실행합니다.

진입점 메서드에서 던진 예외 무시 옵션을 사용하여 진입점 메서드에서 던진 예외를 무시합니다. 진입점 메서드는 다음 검사(에디터 | 검사 | Java | 선언 중복성 | 사용되지 않는 선언)로 구성됩니다.
