기본 long, integer, short 또는 byte 인수에서 새로운 Long, Integer, Short 또는 Byte 객체를 인스턴트화하려는 시도를 보고합니다.

Java 5에 도입된 static 메서드인 valueOf()를 사용하는 것이 좋습니다. 기본적으로 이 메서드는 -128~127 사이 값의 객체를 캐싱합니다.

예:


  Integer i = new Integer(1);
  Long l = new Long(1L);

빠른 수정이 적용된 후 코드는 다음과 같이 변경됩니다:


  Integer i = Integer.valueOf(1);
  Long l = Long.valueOf(1L);

이 검사는 프로젝트 또는 모듈의 언어 수준이 5 이상인 경우 보고합니다.

문자열 인수가 있는 새 숫자 표현식 무시 옵션을 사용하여 String 인수가 있는 숫자 생성자 호출을 무시합니다.

생성자가 @Deprecated인 경우에만 보고 옵션을 사용하여 사용이 중단된 생성자 호출만 보고합니다. JDK 9부터 Long, Integer, Short 및 Byte 생성자는 사용되지 않습니다.
