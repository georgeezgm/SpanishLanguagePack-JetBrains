컬렉션 또는 배열을 반복하며 개선된 for 루프(foreach 반복 구문)로 자동 대체 가능한 루프를 보고합니다.

예:


  for (Iterator<String> iterator = list.iterator(); iterator.hasNext(); ) {
    String item = iterator.next();
    System.out.println(item);
  }

빠른 수정 적용 후:


  for (String item : list) {
    System.out.println(item);
  }

Report indexed 'java.util.List' loops 옵션을 사용하여 list.get(index) 호출을 포함한 루프를 찾습니다. 예를 들면 list.remove(index) 를 호출하여 프로세스의 기준이 되는 리스트를 변경하지 않는 한 일반적으로 이 루프는 강화된 for 루프와 교체할 수 있습니다. 후자의 경우 강화된 for 루프는 ConcurrentModificationException 이 발생할 가능성이 있습니다. 또한 list.get(index) 루프가 약간 빠르게 작동하는 경우가 있습니다.

Do not report iterations over untyped collections 옵션을 사용하여 유형 매개변수가 없는 컬렉션을 무시합니다. 이와 같은 방법으로 java.lang.Object 형의 강화된 for 루프 변수의 생성과 루프 변수를 사용하는 형 변환의 삽입을 방지합니다.

이 검사는 프로젝트 또는 모듈의 언어 수준이 5 이상인 경우에만 보고합니다.
