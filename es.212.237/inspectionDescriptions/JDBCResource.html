안전하게 닫히지 않은 JDBC 리소스를 보고합니다. 이 검사에서 보고되는 JDBC 리소스에는 java.sql.Connection, java.sql.Statement, java.sql.PreparedStatement, java.sql.CallableStatement, java.sql.ResultSet이 포함됩니다.

이 검사는 기본적으로 리소스가 이름에 'close' 또는 'cleanup'을 포함하는 모든 메서드에 의해 닫힐 수 있다고 가정합니다.

예:


  ResultSet findAllElements(Connection connection) throws SQLException {
    PreparedStatement statement = connection.prepareStatement("SELECT * FROM TABLE"); // 구문이 닫히지 않음
    statement.execute();
    return statement.getResultSet();
  }

다음 옵션을 사용해 검사를 구성하세요:

  • JDBC 리소스를 try 블록 내부에서 여는 것을 허용할지 여부를 설정합니다. 이 스타일은 try 블록 앞에서 리소스를 여는 것보다 장황하기 때문에 그다지 바람직하지 않습니다.
  • 리소스가 인수로 전달된 메서드 호출에 의해 리소스가 닫힐 수 있는지 여부를 설정합니다.
