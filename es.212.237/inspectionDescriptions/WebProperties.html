웹 모듈을 서버에 배포하는 데 사용되는 설명자 내 다음과 같은 문제를 보고합니다:
  • 인스턴스화할 수 없는 클래스 참조
  • 필수 클래스를 확장하지 않는 클래스 참조
  • 부적절한 스코프를 가진 클래스 참조
  • 빈 태그 및 속성 값
  • 요구되는 패턴(예: Java 식별자)과 일치하지 않는 태그 및 속성 값
  • 필요한 하위 태그 또는 속성을 포함하지 않는 태그
  • 이름이 중복된 객체를 정의하는 태그

예:



<web-app>
  <filter-mapping>
    <filter-name>CustomFilter</filter-name> // 오류: servlet-name 또는 url-pattern 하위 태그가 정의되어야 함
  </filter-mapping>
  <servlet-mapping>
    <servlet-name>CustomServletName</servlet-name>
    <url-pattern>/ baseUrl</url-pattern> // 오류: URI-pattern은 공백을 포함할 수 없음
  </servlet-mapping>
</web-app>
