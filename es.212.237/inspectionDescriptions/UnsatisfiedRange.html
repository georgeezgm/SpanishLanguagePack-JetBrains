선언된 메서드 반환 범위를 준수하지 않는 메서드에서 반환된 숫자 값을 보고합니다. 몇 가지 어노테이션을 사용하여 메서드 반환 범위를 선언할 수 있습니다:
  • JetBrains 어노테이션 패키지의 org.jetbrains.annotations.Range('from' 및 'to' 지정)
  • Checker Framework 어노테이션 패키지의 org.checkerframework.common.value.qual.IntRange('from' 및 'to' 지정)
  • Checker Framework 어노테이션 패키지의org.checkerframework.checker.index.qual.GTENegativeOne(범위: '>= -1')
  • Checker Framework 어노테이션 패키지의org.checkerframework.checker.index.qual.NonNegative(범위: '>= 0')
  • Checker Framework 어노테이션 패키지의org.checkerframework.checker.index.qual.Positive(범위: '> 0')
  • JSR 305 어노테이션 패키지의javax.annotation.Nonnegative(범위: '>= 0')
  • javax.validation.constraints.Min(최소 값 지정)
  • javax.validation.constraints.Max(최대 값 지정)

예:

  @Range(from = 0, to = Integer.MAX_VALUE) int getValue() {
    // 경고: -1은 선언된 범위에서 벗어남
    return -1;
  }
