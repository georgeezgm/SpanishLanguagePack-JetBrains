final 클래스의 protected 멤버를 보고합니다.

final 클래스는 상속될 수 없으므로 메서드를 protected로 표시할 경우 혼란을 야기할 수 있습니다. 해당 멤버를 private 또는 package-visible로 선언하는 것이 좋습니다.

예:

record Bar(int a, int b) {
  protected sum() { 
     return a + b;
  }
}

빠른 수정 적용 후:

record Bar(int a, int b) {
  sum() { 
     return a + b;
  }
}

예시에서 볼 수 있듯, 클래스는 명시적 혹은 묵시적으로 final로 표시될 수 있습니다.
