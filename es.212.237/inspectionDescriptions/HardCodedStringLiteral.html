하드코딩된 String 리터럴의 모든 인스턴스를 보고합니다.

국제화된 환경에서 하드코딩된 String 리터럴은 오류일 가능성이 높습니다. 이 검사는 빈 문자열 및 공백으로만 구성된 문자열을 보고하지 않습니다. 빠른 수정을 사용하여 문자열 리터럴을 java.util.ResourceBundle.getString() 메서드 호출로 변환할 수 있습니다.

검사가 다음에 포함된 문자열을 보고해야 할지 더 상세히 살펴보는 검사 옵션을 사용합니다.

  • assert 문(예: assert str.equals("message"))
  • 예외 생성자 호출(예: new Exception("message"))
  • JUnit assert 호출(예: assertEquals(str, "message"))
  • 문자열을 반환하는 메서드의 유일한 인수(예: getStringByKey("key"))
  • 기존의 정규 클래스 값이 포함된 리터럴(예: Class.forName("java.lang.Object"))
  • 기존의 정규 프로퍼티 키 값이 포함된 리터럴(예: bundle.getString("authentication.failed"))
