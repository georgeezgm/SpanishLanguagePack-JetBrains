다른 객체의 private 또는 protected 필드를 사용하는 객체를 보고합니다.

Java 에서는 현재의 객체와 동일한 클래스의 객체를 위한 필드를 사용하지만 일부 코드 스타일은 이러한 사용을 추천하지 않습니다. 또한 private 필드의 직접적인 액세스는 프레임워크가 프록시를 사용하여 액세스를 중개할 수 있는 것과 같이 메서드 호출을 통해 다른 객체에 액세스할 것으로 예상하는 Spring 및 Hibernate와 같은 구성요소 지향의 아키텍처에서 실패할 수 있습니다.

예:


public class Base {
    protected int bar;

    class Inside {
        int f(Base base) {
            return base.bar;  // 경고: 비 public 필드에 직접 액세스
        }
    }
}

class SomewhereElse {
    void m(Base base) {
        base.bar = 1; // 경고: 비 public 필드에 직접 액세스
        }
    }
}

검사 구성:

  • 동일한 클래스의 액세스 무시 옵션을 사용하여 동일한 클래스의 액세스를 무시하고 내부 또는 외부 클래스의 액세스를 보고합니다.

    내부 클래스의 액세스도 무시하는 경우 중첩된 내부 클래스의 액세스 무시를 사용합니다.

  • 'equals()' 메서드의 액세스 무시를 사용하여 equals() 메서드의 액세스를 무시합니다.
