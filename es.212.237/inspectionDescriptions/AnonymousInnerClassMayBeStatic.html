static 내부 클래스로 안전하게 대체할 수 있는 익명 클래스를 보고합니다. 익명 클래스는 주변 메서드의 로컬 클래스 또는 둘러싸는 인스턴스를 명시적으로 참조하지 않는 경우 static 내부 클래스일 수 있습니다.

static 내부 클래스는 둘러싸는 인스턴스에 대한 묵시적 참조를 유지하지 않습니다. 이를 통해 메모리 누수의 일반적 원인이 방지되며, 클래스 인스턴스당 메모리 사용량이 감소합니다.

빠른 수정은 익명 클래스를 명명된 static 내부 클래스로 추출합니다.

예:

  
  void sample(){
    Thread thread = new Thread(new Runnable() {
      @Override
      public void run() {
      }
    });
  }

빠른 수정 적용 후:

  
  void sample(){
    Thread thread = new Thread(new Task());
  }

  private static class Task implements Runnable {
    @Override
    public void run() {
    }
  }
