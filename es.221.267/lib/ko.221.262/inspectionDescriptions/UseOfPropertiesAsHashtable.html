java.util.Properties 객체에서 다음 메서드를 호출하는 경우를 보고합니다.
  • java.util.Hashtable#put
  • java.util.Hashtable#putIfAbsent
  • java.util.Hashtable#putAll
  • java.util.Hashtable#get

역사적인 이유로 java.util.Properties는 java.util.Hashtable를 상속받지만 String 외의 타입을 가진 프로퍼티 값의 손상을 방지하기 위해 그러한 메서드는 사용하지 않는 것이 좋습니다.

java.util.Properties#putAll은 java.util.Hashtable#putAll을 재정의하지만 맵 내의 키 매개변수 및 값 매개변수 모두 String 타입인 경우 보고되지 않습니다.

예:


  Object f(Properties props) {
    props.put("hello", "world");
    props.putIfAbsent("hello", "world");
    props.putAll(new HashMap<>());
    return props.get("Hello");
  }

빠른 수정 적용 후:


  Object f(Properties props) {
    props.setProperty("hello", "world");
    props.putIfAbsent("hello", "world");
    props.putAll(new HashMap<>());
    return props.getProperty("hello");
  }
