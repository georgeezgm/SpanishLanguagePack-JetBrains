go.mod 를 수정할 때마다 go list 를 실행하지 않으려면 &shortcut:ShowSettings; 을(를) 눌러 설정을 열고 Build, Execution, Deployment | Build Tools (으)로 이동합니다. Reload project after changes in the build scripts 체크박스를 선택 해제합니다.

기본적으로 GoLand은(는) go.mod 파일이 변경된 후 프로젝트를 다시 로드합니다. 그러나 External changes 옵션을 선택하면 IDE 내에서 파일을 편집할 때 go list가 자동으로 실행되지 않습니다. 이 경우 Go 모듈 변경 내용 로드 아이콘이 표시됩니다. 편집을 완료하고 아이콘을 클릭하여 변경 내용을 로드합니다.
