어노테이션 중 하나가 불안정하다고 표시된 API의 사용 위치를 보고합니다. 그러한 API는 향후 버전에서 변경되거나 제거되어 이를 사용하는 코드에서 오류를 발생시킵니다.

불안정한 API를 표시할 때 사용하는 어노테이션을 아래 표에 나열했습니다.

기본적으로 불안정한 API의 정의가 같은 프로젝트의 소스에 있다면 검사할 때 그러한 API의 사용 위치를 무시합니다. 그러한 경우 API를 변경할 때 사용 위치를 업데이트할 수 있습니다. 하지만 프로젝트가 크다면 이는 불편할 수 있으므로 프로젝트 소스와 라이브러리에서 모두 ㄴ언된 불안정한 API의 사용 위치를 보고하는 이 프로젝트에서 선언한 API 무시 옵션을 끌 수 있습니다.
