Number를 확장하는 클래스의 객체에 대한 toString() 호출을 보고합니다. 이러한 호출은 일반적으로 국제화된 환경에서 올바르지 않으며 일부 로케일 특정 형식을 대신 사용해야 합니다.

예:


  void print(Double d) {
    System.out.println(d.toString());
  }
이 문제를 수정하기 위한 가능한 방법은 다음과 같을 수 있습니다: 

  void print(Double d) {
    System.out.printf("%f%n", d);
  }
이 값은 호스트 환경을 기반으로 JVM을 시작하는 동안 설정된 기본 로케일을 사용하여 숫자의 형식을 지정합니다.
