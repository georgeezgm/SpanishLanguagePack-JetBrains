더 작은 범위로 이동할 수 있는 모든 변수 선언을 보고합니다.

이 검사는 주로 메서드 시작 부분의 Pascal 형식 선언에 유용합니다. 또한 너무 광범위한 범위를 가진 변수도 리팩터링 후에 남는 경우가 많습니다.

sb 이동의 예시:


  StringBuilder sb = new StringBuilder();
  System.out.println();
  sb.append(1);

빠른 수정 적용 후:


  System.out.println();
  StringBuilder sb = new StringBuilder();
  sb.append(1);

검사 구성:

  • 내부 블록으로 이동할 수 있는 변수만 보고 옵션을 사용하여 더 깊은 코드 블록을 이동할 수 있는 변수만을 보고합니다. 예를 들어, 옵션이 활성화되면 sb 변수에는 이동이 제안되지만 다음의 코드에는 제안되지 않습니다. 
    
  StringBuilder sb = new StringBuilder(a);
  if (flag) {
    sb.append(1);
  }
  • new 식을 이니설라이저로 가진 변수 보고(잠재적으로 안전하지 않음) 옵션을 사용하여 new 식으로 초기화되는 변수를 보고합니다. 생성자에 비 로컬 부수 효과가 있는 경우 검사가 잠재적으로 안전하지 않게 됩니다. 예를 들어, 옵션이 활성화되면 foo 변수에 대한 이동이 제안됩니다. 
    
  class Foo {
    static List<Foo> fooList = new ArrayList<>();
    String bar;

    Foo(String bar) {
        this.bar = bar;
        fooList.add(this);
    }

    public static void main(String[] args) {
      Foo foo = new Foo("bar"); // 안전하지 않은 경우에도 이동 가능
      System.out.println(fooList.size());
      System.out.println(foo.bar);
    }
  }
