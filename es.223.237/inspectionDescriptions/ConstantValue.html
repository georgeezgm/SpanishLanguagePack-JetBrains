true, false, null 혹은 0과 같이 결과가 항상 같은 표현식과 조건을 보고합니다. 이러한 표현식은 상응하는 상수 값으로 대체될 수 있습니다. 이런 경우는 코드 내에 버그가 있다는 것을 의미할 확률이 매우 높습니다.

예:

  // 항상 true
  // 원인: ||가  && 대신 사용됨
  if (x > 0 || x < 10) {}

  System.out.println(str.trim());
  // 항상 false
  // 원인: null 체크 전에 변수가 역참조됨
  if (str == null) {}

null 값 가능성을 표시하는 어노테이션, @Contract 어노테이션, @Range 어노테이션 등 여러 어노테이션으로 검사 동작이 제어될 수 있습니다.

검사 구성:

  • 항상 true임이 정적으로 입증된 조건이 포함된 어설션을 보고하지 않음 옵션을 사용하여 항상 true임이 정적으로 입증된 어설션을 보고하지 않습니다. if (alwaysFalseCondition) throw new IllegalArgumentException();와 같은 조건도 이에 포함됩니다.
  • assert 문 무시 옵션을 사용하여 검사에서 assert 문이 취급되는 방식을 제어합니다. 기본적으로 이 옵션은 비활성화되어 있습니다. 즉, 어설션이 실행되는 것으로 간주됩니다(-ea 모드). 이 옵션을 활성화하면 어설션이 완전히 무시됩니다(-da 모드).
  • 상수가 변수에 저장되면 경고 옵션을 사용하여 값이 상수로 알려진 변수가 사용되면 경고를 표시하세요.

IntelliJ IDEA 2022.3 이전에는 이 조사가 "상수 조건 및 예외" 검사의 일부였습니다. 이제는 두 가지 검사, "상수값" 과 "Null 가능성 및 데이터 플로우 문제"로 나뉩니다.
