잘못 배치된 가능성이 있는 Gradle 메서드 호출을 감지합니다.

Gradle buildscripts는 중첩 클로저가 많이 있어서 코드 구조가 마크업 언어와 비슷합니다. Groovy의 특성으로 인해 바깥 클로저에 작성한 메서드는 내부에서도 쓸 수 있습니다. 이런 메서드는 범위 밖에서 작성되면 의미가 없을 수 있습니다.
종종 이런 상황을 감지하기 어려울 수 있습니다. 이 검사의 목표는 그런 메서드를 감지하는 것입니다.

예:


repositories {
  // 'repositories'의 위임자는 'exclusiveContent' 메서드를 갖고 있으며, 여기에 작성될 수 있습니다
  maven {
    // 'maven'의 위임자는 'content', 메서드를 갖고 있으며, 여기에 작성될 수 있습니다
    // 그러나 'exclusiveContent'도 쓸 수 있습니다
    exclusiveContent {} // 'exclusiveContent'를 보고합니다
  }
}
